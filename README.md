# Marshall SBTI Test

一个**纯前端、开箱即用的静态人格测试页面**，主题为「SBTI 人格测试」。项目当前以打包后的产物形式保存（主要是单页 HTML + 两个预构建 JS 依赖），可直接部署到任何静态文件服务器。

---

## 1. 项目定位

本仓库的核心是一个娱乐向人格测试网页，特点：

- 无后端依赖，打开页面即可运行。
- 题目顺序会随机化，支持“补充题”条件触发。
- 根据多维度评分进行匹配，输出最终人格类型与解释。
- 对移动端且非 B 站 App 场景，优先引导用户唤起 B 站打开官方页面。

> 说明：仓库中代码是压缩/打包后的发布版本，不是源代码工程；因此更适合作为“静态页面分发”而不是二次开发基座。

## 3. 功能总览

### 3.1 三屏流程

页面按三个区块组织：

1. `intro`：介绍页（开始测试入口）。
2. `test`：答题页（题目列表、进度条、提交）。
3. `result`：结果页（人格类型、解读、维度明细、海报图）。

通过 `active` 类在三个屏之间切换。

### 3.2 答题机制

- 每题采用 1～7 分（单选）量表。
- 问卷主题目会被随机打散。
- 存在一个“闸门题”逻辑：当 `drink_gate_q1` 选择特定分值（3）时，插入 `drink_gate_q2` 补充题。
- 进度条按“已答 / 当前题数（含动态补充）”实时更新。
- 未完成全部题目时禁用提交按钮。

### 3.3 评分与类型匹配

- 每题归属一个维度（如 `S1/E2/A3/...`）。
- 对维度累积分数后，将每个维度映射为 `L / M / H` 三级。
- 再与类型库中的模式串进行距离计算，得到：
  - 距离（distance）
  - 精确命中数（exact）
  - 匹配度（similarity）
- 默认输出最优匹配人格。
- 包含两个特殊分支：
  - “隐藏人格”触发：直出 `DRUNK`。
  - 低匹配兜底：强制分配 `HHHH`。

### 3.4 结果展示

结果页展示：

- 主类型代码 + 中文名
- 匹配信息与模式说明
- 类型简介与解释文案
- 全维度明细（等级、原始分、解释）
- 人格配图（若对应图片存在）
- 娱乐性免责声明

### 3.5 移动端唤起 B 站

对于“移动端 + 非 BiliApp”的访问：

- 首页按钮文案切换为“去B站测试”。
- 点击后调用 `initOpenAppTag`，尝试唤起 B 站并跳转到指定活动页面。

这意味着：该页面既可独立运行，也可以在特定场景下作为 B 站内体验入口的跳转壳。

## 4. 如何本地运行

本项目无需构建，直接静态托管即可。

### 方式 A：Python（推荐）

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

### 方式 B：Node（如已安装 serve）

```bash
npx serve .
```

## 6. 二次开发注意事项

由于当前仓库为“发布产物形态”，直接修改存在以下成本：

1. **可读性较差**：逻辑多为压缩后单行代码。
2. **变更风险高**：小改动可能影响大片逻辑。
3. **不可追溯**：缺少构建配置与未压缩源码。

如果计划长期维护，建议：

- 将业务逻辑拆回模块化源码（TypeScript/ESM）。
- 把题库、人格库、文案抽离为 JSON 配置。
- 引入 lint/format/test 流程。
- 增加结果计算的单元测试（尤其是特殊人格分支）。
